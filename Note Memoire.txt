SERVEUR Comment arrêter Symfony définitivement sur Windows

Option 1 (PowerShell / CMD)

Ferme le serveur Symfony CLI :
taskkill /F /IM symfony.exe

Puis tue PHP s’il reste :
taskkill /F /IM php-cgi.exe

Vérifie :
symfony server:status
Tu devrais voir :
The web server is not running

✅ Là le serveur est vraiment arrêté et il ne redémarrera pas tout seul.



Parfait 👍 Voici une commande PowerShell “tout en un” pour Windows qui va arrêter complètement Symfony CLI + PHP, et vérifier le statut final, sans qu’il redémarre tout seul :

# Tout en un : stop Symfony, kill PHP, vérifier statut
taskkill /F /IM symfony.exe -ErrorAction SilentlyContinue; taskkill /F /IM php-cgi.exe -ErrorAction SilentlyContinue; symfony server:status

🔹 Explications :
taskkill /F /IM symfony.exe → tue Symfony CLI, le vrai “chef” qui relance PHP.
taskkill /F /IM php-cgi.exe → tue PHP si un processus traîne.
-ErrorAction SilentlyContinue → évite les messages d’erreur si le processus n’existe pas.
symfony server:status → te montre que tout est bien arrêté.
